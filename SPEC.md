# SPEC — FPPC Tuned Embeddings (draft)

> **Status:** Draft. Anchored in Check 1 (complete) and Check 2 (partially
> complete — bge-base + OpenAI replay only; nomic-v1.5 and bge-large pending
> on Strix Halo). The "target metric" section will firm up once we have the
> full open-model baseline picture.

## Problem

The FPPC's 14,096 advisory and formal opinions are the primary reference
corpus for analyzing conflict-of-interest questions involving California
public agencies. The production search engine
(`../fppc-opinions-app`, derived from search-lab Experiment 009) is a
BM25-dominated hybrid using OpenAI `text-embedding-3-small` as the semantic
component. Within that hybrid:

- Pure BM25 alone scores **nDCG@5 = 0.358** on the 65-query eval set.
- Pure semantic (OpenAI) alone scores **nDCG@5 = 0.262**.
- The fused hybrid scores **nDCG@5 = 0.387, MRR = 0.684**.

The semantic axis is the weaker partner. The intuition driving this project:
the corpus contains roughly 10,000+ naturally-occurring `(question, opinion)`
pairs (where modern opinions have explicit `Question`, `Conclusion`, `Analysis`
sections, and older opinions had `question_synthetic` generated by an LLM).
That's the cleanest kind of training data for contrastive embedding fine-tuning:
the question is exactly what a user would search for; the opinion is exactly
the document we want retrieved.

## Hypothesis

A domain-adapted open-weights embedding model, fine-tuned on
`(opinion_question, opinion_text)` pairs (plus mined hard negatives and
optional augmentations), will:

1. **Outperform `text-embedding-3-small` on semantic-only retrieval** for the
   eval set — i.e., lift nDCG@5 of the semantic-only path from ~0.26 toward
   ~0.35–0.40.
2. **Provide additive lift in the production hybrid** — i.e., move Experiment
   009's combined nDCG@5 above the current 0.387 ceiling. (This is the
   uncertain claim: BM25 may already capture most findable signal.)

Both outcomes are informative. Even a "semantic doubles, hybrid barely
moves" result is a publishable negative for future legal-corpus work.

## Non-goals (for v1)

- Replacing the BM25 path or the citation-routing logic from Experiment 009.
- Building a new search engine architecture; we plug the tuned model into
  the existing one.
- Training a reranker (could be a follow-on project).
- Fine-tuning a generative model. This is strictly about the embedder.
- Productionizing. If results justify it, we hand off to the app repo.

## Target model

**Tentatively** `BAAI/bge-base-en-v1.5` (110M, 768d). Justification:

- Check 2 baseline shows it matches `text-embedding-3-small` out of the box
  (nDCG@5 = 0.268 vs 0.263; MRR = 0.478 vs 0.494). No quality cost for moving
  off OpenAI.
- Mature fine-tuning support via `sentence-transformers`.
- 110M params is comfortable for the Strix Halo (128GB unified) — full
  fine-tune feasible, multiple short runs per day plausible.

Re-evaluate after `nomic-embed-text-v1.5` (and optionally `bge-large-en-v1.5`)
finish on Strix Halo. If nomic significantly outperforms BGE-base on the
baseline (especially on long-doc topics), reconsider.

## Success criteria

To be finalized after Check 2 completes. **Provisional** targets, each
measured on the 65-query eval set:

- **Threshold (must hit)**: semantic-only nDCG@5 ≥ 0.30 (clearly above the
  OpenAI/bge-base baseline of ~0.26–0.27).
- **Goal**: semantic-only nDCG@5 ≥ 0.35 (approaches BM25-only quality with
  just embeddings).
- **Stretch**: hybrid (Experiment 009 with tuned embeddings) nDCG@5 ≥ 0.42
  and MRR ≥ 0.72.
- **Subgroup**: meaningful (≥ 5 absolute points) improvement on the
  `conflicts_of_interest` topic (current MRR 0.415, the corpus's hardest
  topic).

## Constraints

- **Eval integrity**: 596 eval-referenced opinions (4.23% of corpus) are
  excluded from training entirely — text, real questions, synthetic
  questions. See `notes/check1_leakage_report.md` for full reasoning.
- **Citation hardware**: experiments run on Nick's Strix Halo
  (Ryzen AI Max+ 395, 128GB unified). PyTorch + sentence-transformers; ROCm
  if usable, else CPU.
- **No re-OCR / re-extraction**: trust the corpus JSONs as-is.

## Open questions to answer before training

- Whether `question_synthetic` (LLM-generated questions for older opinions)
  is useful training data or noise. Could be ablated.
- Whether citation-graph signals (`citations.cited_by`, `prior_opinions`) can
  yield useful weak-supervision triplets.
- How aggressively to mine hard negatives from BM25 vs random negatives.
- Whether augmentation (LLM-paraphrased queries in multiple registers) is
  necessary or net-negative.
